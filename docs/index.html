The Problem With AI

AI is powerful but inconsistent. Every conversation feels like the first one. You spend more time managing context than building.

🔄

Loses Context

Between sessions, AI forgets decisions, patterns, and constraints. You start over every time.

📈

Over-Engineers

No scope boundaries. AI adds features, optimizes prematurely, and bloats your MVP.

🎯

Goes Off Track

No priority system. AI jumps between concerns instead of shipping a focused MVP.

The Solution

AI OS brings structure, discipline, and execution to AI-assisted development.

🗂️

Institutional Memory

MEMORY.md + VERSION.md track decisions, patterns, and project evolution. AI remembers.

🎯

Scope Enforcement

CONTEXT.md + STATE.md define boundaries. AI can't add scope without permission.

👥

Multi-Agent System

10 specialized agents (planner, architect, engineer, reviewer, etc.). Right agent for each job.

⚙️

Quality Gates

Automatic reviews, security checks, and issue blocking. Code doesn't merge if CRITICAL issues remain.

📊

Execution Workflows

Feature loops, bug fixes, deployments. Work happens in predictable, repeatable sequences.

⚡

MVP-First Discipline

Optimizer locked until post-MVP. Perfection deferred. Shipping prioritized.

How It Works

1

Define Your Project

Fill CONTEXT.md with your project scope, tech stack, and constraints.

# CONTEXT.md
## Project
TodoAPI — REST API with JWT auth, Postgres
## Features (MVP)
- User registration + login
- Create/read/delete todos
2

Track Your State

Use STATE.md to track current phase, focus, and completed work.

# STATE.md
## Current Phase
Foundation (DB + auth setup)
## Current Focus
User model + migrations
3

Activate Agents

AI OS automatically selects the right agents for your project. Planner → Architect → Engineer → Reviewer → Deploy.

✅ Active Agents:
planner, architect, engineer, reviewer
backend, database, api-designer, security
devops (enabled post-MVP)
4

Execute With Discipline

Run commands like /plan, /build, /review. Quality gates are automatic.

→ /plan
Sequences your work
→ /build
Implements current task
→ /review
Gates quality (PASS/BLOCKED)
