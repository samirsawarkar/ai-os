Boot Sequence (6 Steps) + Authority Hierarchy

BOOT SEQUENCE (Every Session): 1. Read CONTEXT.md (project definition + scope lock) 2. Read STATE.md (current phase + current focus) 3. Read PROJECT_PLAN.md (if exists) 4. Select active agents (auto-select based on task) 5. Confirm Current Focus (from STATE.md) 6. Execute ONLY current task AUTHORITY HIERARCHY (Conflict Resolution): 1. CONTEXT.md → absolute authority (scope boundaries) 2. STATE.md → current focus overrides suggestions 3. planner → controls scope + sequence 4. reviewer → can BLOCK with CRITICAL issues 5. security → can BLOCK unsafe implementations 6. engineer → executes only approved tasks 7. optimizer → lowest priority (post-MVP only) Hard Rules: • Inactive agents = zero authority • Agents cannot override higher layer decisions • If conflict → follow hierarchy strictly

3. Core Concepts & Files

📋 CONTEXT.md (Project Definition + Scope Lock)

Purpose:

Single source of truth. SCOPE LOCK prevents creep. AI reads every session. Mandatory enforcement.

## SCOPE LOCK (MANDATORY) * This file defines what gets built. * If a request is not in CONTEXT.md, it does NOT get built. * Agents cannot expand scope without explicit user approval. ## Project Name TodoAPI ## Description REST API with JWT auth, PostgreSQL ## Features (MVP - FIXED) - User registration + login - Create/read/delete todos - Deploy to Railway ## Tech Stack FastAPI, PostgreSQL, JWT, Railway ## Constraints - Solo developer - MVP in 5 days - $0 spend

📊 STATE.md (Live Execution State)

Purpose:

Updated EVERY agent action. Tracks phase, focus, completed work. Essential for continuity.

## Current Phase Foundation (DB + Auth) ## Current Focus Implement User model + migrations ## Task Priority HIGH ## Completed ✅ Project setup ✅ DB connection ## Pending - User model + migrations - Register/login endpoints - Security review ## Known Issues None

🤖 AGENTS.md (Execution Engine & Routing)

Purpose:

Master execution engine. 6-step boot sequence + 7-level authority hierarchy for conflict resolution.

EXECUTION ENGINE: Boot: 6 steps (CONTEXT → STATE → PROJECT_PLAN → Auto-select → Confirm Focus → Execute) Authority Hierarchy (Conflict Resolution): 1. CONTEXT.md (scope authority - absolute) 2. STATE.md (current focus - overrides suggestions) 3. planner (scope + sequence) 4. reviewer (BLOCK on CRITICAL issues) 5. security (BLOCK on unsafe code) 6. engineer (execute only approved tasks) 7. optimizer (lowest priority, post-MVP only) Auto-Activation: /plan → planner /build → engineer + domain agents /review → reviewer + security /debug → debugger

✅ EXECUTION_RULES.md (Quality Discipline)

Purpose:

Enforces quality. No task completes without PASS verdict.

6-POINT QUALITY GATE (ALL MUST PASS): ✓ Minimal - No unnecessary code ✓ Correct - Logic verified + tested ✓ Necessary - All features from CONTEXT.md ✓ Scalable - Performance acceptable ✓ Compliant - Security + auth passed ✓ Simple - Easy to maintain RESULT: PASS (merge) or BLOCKED (fix + retry)

🚀 START_SESSION_PROMPT.md (Session Boot)

Purpose:

Session initialization. Copy → paste into AI chat to boot system every time.

Boot AI-OS. 1. Read /CONTEXT.md (project definition) 2. Read /STATE.md (current focus) 3. Read /PROJECT_PLAN.md (if exists) Confirm: - Project: [name from CONTEXT] - Current Focus: [from STATE.md] - Active Agents: [auto-selected] Ready. What's next?

🧠 MEMORY.md

Tracks: Decisions, patterns, mistakes, constraints

Updates: After every session (mandatory step 11)

📌 VERSION.md

Tracks: Evolution (v0.1.0 → v1.0.0)

Updates: When logic changes (mandatory step 11)

5. Integration: VS Code & Cursor

Cursor: Optimal Integration

Why Cursor:

  • ✓ @workspace context (@CONTEXT.md @STATE.md)
  • ✓ Chat persists in project folder
  • ✓ Creates/edits files directly
  • ✓ Terminal integration
  • ✓ Best for AI OS workflows

Setup (5 Minutes):

# 1. Clone AI OS git clone https://github.com/samirsawarkar/ai-os . # 2. Open Cursor cursor . # 3. Create new chat (Cmd+L) # 4. Paste: @workspace Read @CONTEXT.md and @STATE.md Follow boot sequence from @AGENTS.md Then execute: /plan

Workflow (Ongoing):

@workspace /plan "feature name"
@workspace /build
@workspace /review
@workspace /debug "bug description"

VS Code + Copilot

Setup:

  • 1. Install GitHub Copilot Chat extension
  • 2. Clone AI OS files into your project
  • 3. Open Copilot Chat (Cmd+Shift+I)

Command Format:

@workspace Read @CONTEXT.md and @STATE.md Boot sequence from @AGENTS.md: 1. Read all config files 2. Activate agents 3. Execute /plan Start with /plan

Note: VS Code has less context than Cursor. May need to paste AGENTS.md manually.

Claude Web (No Editor)

Setup:

  • 1. Go to claude.ai
  • 2. Create new conversation
  • 3. Paste your project files

Initial Message:

I am using AI OS framework. Here is my project setup: [PASTE CONTEXT.md] [PASTE STATE.md] [PASTE AGENTS.md] Follow boot sequence and run: /plan

Workflow: Copy output → Paste to STATE.md → Save locally → Paste in next chat
