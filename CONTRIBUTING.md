# Contributing to AI OS

Thank you for your interest in contributing! AI OS is a community-driven project.

## How to Contribute

### 1. Report Bugs
- **Create an issue** on GitHub with:
  - Clear title and description
  - Steps to reproduce
  - What you expected vs. what happened
  - Your project type (REST API, full-stack, etc.)

### 2. Suggest Improvements
- **Open a discussion** for:
  - New agent types
  - Workflow improvements
  - Better documentation
  - New features

### 3. Submit Code
- **Fork the repository**
- **Create a feature branch** (`git checkout -b feature/amazing-thing`)
- **Make your changes**
- **Write clear commit messages**
- **Push to your fork**
- **Open a pull request** with description

### 4. Improve Documentation
- Fix typos
- Clarify confusing sections
- Add examples
- Translate to other languages

## Code Style

### Markdown
- Use clear headers
- Link to related files
- Include code examples
- Keep sentences concise

### YAML in Files
- Use 2-space indentation
- Be explicit with true/false values
- Comment complex sections

### Example Files
- Test with a real project
- Show before/after
- Include all necessary steps

## Commit Messages

```
Title: one line, clear and present tense

Description (if needed):
- What changed
- Why it changed
- Any breaking changes
```

Examples:
```
Add devops agent for Kubernetes
Fix boot sequence timing issue
Improve CURSOR_GUIDE.md examples
```

## PR Process

1. **Title** should be clear: `Add PostgreSQL schema examples`
2. **Description** should explain: what, why, how
3. **Reference** any related issues: `Closes #123`
4. **Test** your changes with a real project
5. **Wait for review** — maintainers will provide feedback
